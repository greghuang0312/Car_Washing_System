> 创建日期：2026-03-04  
> 状态：已确认

---

## 项目概述

一个基于微信小程序 + 云开发的自助洗车管理系统。客户扫码开始/结束洗车并支付费用，业主可设置计费标准和查看订单。

## 技术决策总结

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 技术栈 | 微信小程序 + 云开发 | 零服务器成本，运维最简 |
| 硬件控制 | 预留接口，暂不实现 | 先验证业务逻辑 |
| 支付方式 | 模拟支付 | 无需商户号，后续一键替换 |
| 计费方式 | 按分钟线性计费 | MVP 最简，后续可扩展阶梯/套餐 |
| 业主端 | 同一小程序，密码登录 | 一套代码，开发量最小 |
| 管理员创建 | 手动数据库添加 | 避免注册逻辑复杂度 |
| 多站点 | 支持（2 个测试站点） | 数据模型已预留 stationId |

## 数据模型

### `orders` 订单表

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `openid` | string | 客户微信唯一标识 |
| `stationId` | string | 洗车站编号（`station_01` / `station_02`） |
| `status` | string | `washing` / `completed` / `paid` |
| `startTime` | timestamp | 开始洗车时间 |
| `endTime` | timestamp | 结束洗车时间 |
| `duration` | number | 使用分钟数 |
| `amount` | number | 应付金额（单位：分） |
| `paidAt` | timestamp | 支付时间 |

### `settings` 设置表

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 固定 `"default"` |
| `pricePerMinute` | number | 单价（单位：分/分钟） |
| `maxDuration` | number | 最大时长上限（分钟），默认 120 |
| `updatedAt` | timestamp | 最近修改时间 |

### `admins` 管理员表

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `openid` | string | 业主微信唯一标识 |
| `password` | string | 登录密码（哈希） |
| `name` | string | 业主名称 |

> 金额全部以 **分（整数）** 存储，界面上以 **元** 显示，代码自动转换。

## 用户流程

### 客户端

```
扫码（携带 stationId 参数）→ 首页
  │
  ├─ 无进行中订单 → 显示"开始洗车" → startWash()
  │   → 创建订单(status=washing) → 跳转计时页
  │
  └─ 有进行中订单 → 显示"结束洗车" → endWash()
      → 计算时长和费用 → 跳转支付页
      → 模拟支付 → 更新订单(status=paid)

首页底部："我的订单" → 历史订单列表
```

### 业主端

```
首页底部 → "管理入口" → 输入密码登录
  │
  管理面板（Tab 切换）：
  ├─ 费用设置：输入单价（元/分钟）→ 保存
  ├─ 订单列表：按站点筛选，按时间排序
  └─ 数据统计：今日/本周/本月/累计（订单数 + 收入）
```

## 异常处理

| 场景 | 处理 |
|------|------|
| 客户忘记结束洗车 | 超过 maxDuration（默认120分钟）自动结束，按上限计费 |
| 客户结束后不支付 | 订单标记 `completed`，业主可在订单列表看到未支付订单 |
| 同一客户重复扫码 | 检测到进行中订单，显示"结束洗车"而非创建新订单 |
| 业主密码输错 | 提示错误，不限制次数 |
